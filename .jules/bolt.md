## 2024-05-30 - Missing Index on Keyset Pagination Timestamp
**Learning:** The public feed uses keyset pagination via `cursor` and `limit`, ordering by `Grievance.created_at.desc(), Grievance.id.desc()`. However, `created_at` lacked an index. In PostgreSQL, querying an unindexed column with an ORDER BY and a LIMIT without filtering first forces a full table scan and an in-memory or on-disk sort (O(N log N)), which becomes a severe bottleneck as the `grievances` table grows.
**Action:** When implementing or observing keyset pagination (or any frequent sort + limit query) in SQLAlchemy models, ensure the sorted columns have `index=True` to allow O(1) B-Tree lookups.
