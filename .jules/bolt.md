## 2024-05-29 - Keyset Pagination Test Flakiness
**Learning:** When writing tests for keyset pagination endpoints that sort by `created_at` and `id`, bulk inserting test records will create identical timestamps. Because `id` is a UUID, the ordering becomes non-deterministic, causing strict index-based assertions (like `data[0]['title'] == 'Expected Title'`) to fail randomly.
**Action:** Always use set-based assertions (e.g., checking if expected titles exist in a set of retrieved titles) or explicitly manipulate the `created_at` timestamp during test data creation (e.g., adding `timedelta(seconds=i)`) to ensure deterministic ordering.
